# 新增协议消息流程

> 当需要新增客户端-服务端通信消息时，使用此流程。

---

## 步骤 1：阅读协议规范

```
查看 xxz/docs/specs/protocol.md
```

## 步骤 2：设计消息结构

在 `protocol.md` 中添加消息定义，包括：

1. **消息ID** (MsgID)
2. **消息名称** (按命名规范)
3. **字段列表** (类型 + 名称 + 说明)
4. **方向** (C2S / S2C / Bidirectional)

## 步骤 3：实现消息处理

1. 在 `xxz/src/Common/Protocol/` 中添加消息定义
2. 实现发送端打包逻辑
3. 实现接收端解包和处理逻辑

## 步骤 4：同步文档

| 文档 | 更新内容 |
|------|----------|
| `protocol.md` | 消息定义 |
| `interfaces.md` | 相关处理接口 |
| `PROGRESS.md` | 任务完成状态 |
